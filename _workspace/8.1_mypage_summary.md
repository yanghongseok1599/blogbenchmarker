## 아키텍처 결정

### 1. Repository 경유 원칙
- UI (`mypage.js`, `mypage-tab.js`) 는 **supabase 클라이언트를 직접 호출하지 않는다**. `getProfile`, `getMonthlyUsage`, `getActivePlan` 등 repo 함수만 사용.
- 새 repo 2개의 함수 시그니처는 CLAUDE.md의 "DB 접근은 `lib/repositories/` 계층에 추상화" 규칙과 기존 `user-repo.js` 패턴을 그대로 따름.

### 2. 진실의 원천 분리
- **현재 플랜 표시**: `profiles.plan` 우선 (verify-subscription Edge Function 이 쓰기 주체).
- **만료일 계산**: `subscriptions.ends_at` (status IN ('active','cancelled') AND (ends_at IS NULL OR ends_at > NOW()) 조건).
- **결제 수단 표시**: `subscriptions.gateway` (toss/portone).
- 둘이 불일치할 수 있으므로 (구독 만료 직후 webhook 반영 지연 등) 각각 독립적으로 조회하고 UI 에서 매핑.

### 3. 부분 실패 허용
- `mypage.js` / `mypage-tab.js` 모두 `Promise.allSettled` 로 5/4 RPC 병렬 호출. 하나가 실패해도 나머지는 렌더. 실패 항목은 "일부 데이터를 불러오지 못했습니다: 프로필, 구독" 식으로 사용자에게 한국어 알림.

### 4. 만료 배너 dismiss 정책
- `chrome.storage.local.__bbm_expiry_banner_dismissed = { endsAt, dismissedAt }`.
- 키로 `endsAt` 값을 사용하므로 **재결제/연장으로 endsAt 이 바뀌면 자동 재표시**. 재구독 후에도 조용히 사라진 채 방치되는 UX 버그 방지.
- `isExpired === true` 일 때는 dismiss 버튼 숨김(계속 경고 노출).

### 5. 게이지 상태 구간
- `percent < 50`: good(녹색)
- `50 ≤ p < 80`: fair(하늘)
- `80 ≤ p < 100`: warn(주황)
- `p ≥ 100`: danger(빨강)
- `plan === 'unlimited'`: gauge 자체 숨기고 "무제한" 표기로 치환.

### 6. mypage-tab 템플릿 로드
- `panel.html` 의 탭 div 는 정적이고, 실제 마크업은 `mypage-tab.html` 조각.
- `mount()` 에서 `fetch(chrome.runtime.getURL(...))` → `DOMParser` → `document.importNode` 로 노드 주입. **innerHTML 미사용** — DOMParser 는 파싱 단계에 스크립트를 실행하지 않고, fetch 대상이 자체 extension 자원(고정 경로)이므로 XSS 벡터 아님.
- 대안(정적 import) 은 HTML → JS 템플릿 문자열이 필요해 오히려 가독성 떨어짐.

### 7. WAR(web_accessible_resources)
- 사용자 지시로 `mypage/mypage.{html,js,css}` 를 WAR 로 노출.
- **최소 노출 원칙**: `matches` 를 `https://blog.naver.com/*` 한 줄로 고정(host_permissions 범위 내 최협소 매칭).
- 참고: `chrome.tabs.create({ url: chrome.runtime.getURL('mypage/mypage.html') })` 는 extension 컨텍스트에서 WAR 없이도 열리지만, 향후 content script 에서 iframe/링크로 마이페이지를 여는 시나리오 대비한 선제적 선언.

## 계약 / 후속 Phase 대기

### Phase 8.2 결제 연동 (결제 CTA placeholder)
- `mypage.js:openPricingTab()` 와 `expiry-banner.onUpgrade` 콜백은 현재 `chrome.tabs.create('mypage/mypage.html#pricing')` 로 임시 연결.
- 결제 게이트웨이(토스/포트원) 결정 후 `#pricing` anchor 를 실 결제 URL 또는 전용 `pricing.html` 로 교체 필요.

### `profiles.plan` 업데이트 경로
- `verify-subscription` Edge Function 이 `profiles.plan` 을 갱신한다는 전제. 클라이언트(본 Phase)에서는 `plan` 쓰기 없음.

### panel.js 탭 라우팅 (기존 코드)
- panel.js 가 기존 `tabs/{id}-tab.js` 의 `mount(container)` 를 동적 import 하여 활성화한다는 관례에 맞춤. `TAB_IDS` 에 `'mypage'` 이미 존재(panel.js:4).
- 사이드패널 내 "마이" 탭 활성화 시 panel.js 가 `import('./tabs/mypage-tab.js').then(m => m.mount(container))` 호출 필요. 아직 panel.js 가 이 패턴을 사용하지 않는다면 추후 연결점 생성 필요(본 작업 범위 외).

### Phase 10 i18n
- 하드코딩 한국어 라벨("프로필", "현재 플랜", "무료/PRO/무제한" 등) → `chrome.i18n.getMessage()` 로 이관 예정.
