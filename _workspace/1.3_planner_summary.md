## 주의사항 (ARCHITECTURE.md 와의 정합성)

1. **host_permissions 범위**: 본 작업은 사용자 지시대로 `*://*.naver.com/*` 사용. ARCHITECTURE / chrome-extension-security 스킬은 `blog.naver.com/*` 등 더 좁은 범위를 권장. Phase 2 에서 content script 붙일 때 축소 검토 필요.
2. **activeTab + host_permissions 동시 사용**: 스킬은 둘 중 하나만 권장하나 사용자 지시대로 둘 다 포함. Chrome Web Store 심사 전에 실제 사용 코드 기반으로 재평가 권장.
3. **탭 이름 매핑**: 사용자 요청 `분석/벤치마크/생성/마이` ↔ ARCHITECTURE 의 `tabs/` 폴더(`analyze/generate/benchmark/settings`). `settings` 대신 `mypage` 채택 (요청 반영). Phase 5 에서 `tabs/mypage-tab.js` 로 배치.
