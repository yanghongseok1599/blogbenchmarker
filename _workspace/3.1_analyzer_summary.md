## 점수 배분 (합계 100)

사용자가 정의한 5개 섹션에 점수 할당을 확정:

| 섹션 | 만점 | 세부 항목 |
|------|-----|----------|
| `titleSeo` | 20 | 길이(8) + 키워드(6) + 숫자/브래킷(4) + 감점(0 or -2) |
| `contentSeo` | 30 | 문단(8) + 이미지(8) + FIRE-IRE(8) + 신뢰성(6) |
| `hookScore` | 15 | hook-detector confidence 기반 (15 / 10 / 5 / 3 / 0) |
| `readability` | 20 | 평균 문장 길이(12) + 문장 유형 분포(4) + 이모지(4 or -4) |
| `keywordDensity` | 15 | 최상위 키워드 밀도(8) + 제목-본문 일치(7) |
| **합계** | **100** | |

> SKILL.md §1 의 7개 항목(15+15+15+15+15+15+10)을 사용자 정의 5섹션(20+30+15+20+15) 으로 재편성.
> `first impression` 은 `hookScore`, `structure`+`images`+`trust`+`tags` 는 `contentSeo` 로 통합. FIRE 의 First 는 hookScore 로 분리해 더블 카운트 방지.

## Hook Detection 개선 (기존 버그 재현 방지)

| 항목 | 기존(`blog-booster-pro/content/analyzer.js:469-489`) | 신규 (`hook-detector.js`) |
|------|-------------------------------------|------------------------|
| 판정 방식 | 단일 키워드 매칭 | **최소 2개 조건 충족 필수** (§3-2) |
| "안녕하세요? 오늘은…" | 질문형 오판 + 인사형 오판 | QUESTION_ENDING 있으면 GREETING 감점, GREETING 있으면 QUESTION 감점 → 양쪽 모두 match 안 됨 → `unknown` |
| 동률 처리 | N/A (boolean) | TYPE_PRIORITY 로 `statistic > question > storytelling > exclamation > greeting` 우선 |
| 결과 타입 | `type: string` | `{ type, confidence(0~1), firstSentence, matchedPattern, reasons[] }` |
| confidence < 0.5 | 그대로 타입 반환 | `unknown` 으로 강등 |

**false positive 회피 케이스:**
- "안녕하세요? 오늘은 SEO 이야기를 해볼게요." → `unknown` (question 조건 부족 + greeting 조건 부족)
- "3가지만 기억하세요!" → `exclamation` match (느낌표 + 감탄부사 X → confidence 0.5 로 표시)
- "50% 할인이 시작되었습니다." → `statistic` match (수치 + 한국어 명사)
- "저는 지난 주 처음 써봤어요." → `storytelling` match (과거시제 + 1인칭 + 시간표지)

## ReDoS 안전성

모든 정규식이 다음 규칙 준수:
- **앵커**: `$`, `^`, `\b` 명시
- **반복 상한**: `{1,6}`, `{2,80}`, `{2,20}` 등 상한 고정
- **중첩 수량자 없음**: `(a+)+`, `(\w+)*` 형태 0건
- **alternation**: 리터럴 문자열 기반 (역추적 없음)

감사 결과 — 신규 5개 파일의 모든 정규식:
```
nlp-utils.js:     JOSA, EMOJI_RE, VERB_ENDING, ADJ_ENDING, NUMERIC_UNIT_RE, 문장분리 lookbehind
hook-detector.js: QUESTION_ENDING, INTERROGATIVE, SECOND_PERSON, EXCLAMATION_ENDING,
                  EMOTIONAL_ADVERB, STATISTIC_PATTERN, KOREAN_NOUN, PAST_TENSE,
                  FIRST_PERSON, TIME_MARKER, GREETING
seo-analyzer.js:  TITLE_HAS_NUMBER{1,4}, TITLE_HAS_BRACKET, TITLE_BAD_SYMBOLS{2,3},
                  CONTENT_EXAMPLE_CUE, CONTENT_QUOTE_CUE, CONTENT_FIRST_PERSON,
                  CONTENT_URL_RE{1,200}, HEADING_MARKER
extractor.js:     공백/줄바꿈 정리 정규식만 — 전부 단순.
```
→ 10,000자 공백/반복 문자 입력 시에도 선형 시간.

## 경계 케이스 처리

| 케이스 | 동작 |
|-------|------|
| 빈 입력 (`title` / `content` 모두 비어있음) | `buildEmptyResult()` → totalScore 0, recommendations 1개, warnings: `['empty']` |
| 이미지만 있는 글 (charCount < 100, images > 0) | warnings: `['too_short', 'image_only']` |
| 초단문 (charCount < 100) | warnings: `['too_short']` + 제한된 점수 |
| 초장문 (charCount > 10000) | warnings: `['too_long']` + 정상 점수 (hook-detector 는 첫 문단만 스캔) |
| 이모지 폭탄 (density > 5%) | warnings: `['emoji_bomb']` + readability -4 |
| 본문 못 찾음 (extractor) | `runAnalysis` → `{ ok: false, code: 'content_not_found' }` |
| 분석기 예외 | `{ ok: false, code: 'analyzer_exception' }` |

## 네이버 에디터 버전 호환

`extractor.js` 의 셀렉터 체인 — 순서대로 시도 후 첫 match 채택:

| 에디터 | 제목 셀렉터 | 본문 셀렉터 | `detectEditorVersion()` |
|-------|-----------|------------|----------------------|
| SmartEditor ONE | `div.se-title-text` | `div.se-main-container` | `'smarteditor-one'` |
| 구버전 iframe | `div.pcol1 h3 span.itemSubjectBoldfont` | `div#postViewArea` (iframe#mainFrame 내부) | `'legacy'` |
| 레거시 | `span.pcol1` | `div.post_ct` | `'post-ct'` |
| fallback | `meta[property="og:title"]` | — | `'unknown'` |

iframe 내부 접근: `getBlogDocument()` 가 `contentDocument` 를 반환. cross-origin 시 상위 document 로 폴백.

**innerHTML 사용 0건.** 본문 텍스트 추출 시 `cloneNode(true)` 후 블록 요소에 `insertAdjacentText('afterend', '\n\n')` 로 단락 구분 주입. 원본 DOM 미변경.
