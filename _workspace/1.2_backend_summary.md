## 적용된 규칙 체크리스트

- [x] 모든 public 테이블 `ENABLE ROW LEVEL SECURITY`
- [x] FK cascade/set null 명시 (전부 `ON DELETE CASCADE`)
- [x] 모든 timestamp 컬럼 `TIMESTAMPTZ DEFAULT NOW()`
- [x] 각 SQL 파일 상단 한국어 목적/참조 주석
- [x] 각 SQL 파일 하단 `-- ROLLBACK:` 역순 주석
- [x] 쓰기 가능 정책에 `WITH CHECK` 명시
- [x] `is_admin` 관리자 판정 사용 (이메일 비교·JWT 클레임 파싱 금지)
- [x] `handle_new_user()` `SECURITY DEFINER` + `search_path = public` 고정
- [x] `CREATE ... IF NOT EXISTS` / `DROP POLICY IF EXISTS` 로 Idempotent 보장
- [x] 주요 조회 패턴에 인덱스 (user+created_at, blog+fetched_at, GIN on JSONB/array)

## 사용자가 실행할 명령

```bash
# 프로젝트 루트에서
cd "/Users/seok/AI PROJECT/chrome/BLOG BenchMarker"

# 마이그레이션 적용
supabase db push

# 또는 로컬 초기화부터 다시
supabase db reset
```

적용 후 검증 쿼리:
```sql
-- 1. RLS 누락 테이블 확인 (결과 0건이어야 함)
SELECT tablename FROM pg_tables
WHERE schemaname = 'public' AND rowsecurity = false;

-- 2. 테이블별 정책 개수 확인
SELECT tablename, COUNT(*) AS policy_count
FROM pg_policies
WHERE schemaname = 'public'
GROUP BY tablename ORDER BY tablename;

-- 3. 트리거 존재 확인
SELECT tgname FROM pg_trigger
WHERE tgname = 'on_auth_user_created';
```
