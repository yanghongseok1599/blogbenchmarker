## 3. 핵심 설계 결정

### 3.1 RSS 우선 / HTML 폴백
- 1차: `https://blog.naver.com/rss/{blogId}` (RSS XML)
- 폴백: `https://m.blog.naver.com/{blogId}` (HTML 메인)
- 양쪽 모두 `AbortController` + 15초 timeout. 첫 단계 실패 또는 빈 결과면 두 번째 단계 시도.
- 양쪽 collector 모두 host 화이트리스트(`blog.naver.com` / `m.blog.naver.com` / `rss.blog.naver.com`)로 fetch URL 검증.

### 3.2 DOMParser 사용 규칙 (chrome-extension-security)
- RSS: `parser.parseFromString(xml, 'text/xml')` + `getElementsByTagName('parsererror')` 검사 → 실패 시 즉시 throw.
- HTML: `parser.parseFromString(html, 'text/html')` + `querySelectorAll('a[href]')` + `toAbsoluteNaverUrl()` 화이트리스트 정규화.
- 두 경로 모두 DOM 에 직접 삽입하지 않으므로 `<script>` 실행 위험 없음.
- 외부 라이브러리(fast-xml-parser 등) 미사용.

### 3.3 content_snippet XSS 방지
- `stripHtml(input)` 정규식 — 주석 / 태그 / `&nbsp; &amp; &lt; &gt; &quot; &#39;` 엔티티 제거 + 공백 정규화.
- 결과는 1500자로 clamp 후 DB 저장.
- DOM 삽입 경로가 아닌 **저장 단계** 에서 평문화 — UI 가 textContent 로 출력하면 이중 안전.

### 3.4 레이트 리밋 + 동시 실행 락
- 블로그당 최소 10분(`MIN_SYNC_INTERVAL_MS`). chrome.storage.local key `bm_last_sync_{blogId}` 로 추적.
  - DB 컬럼 추가가 불가능(supabase/ 미수정 규칙)해 storage 캐시 사용.
- 동시 실행 락: `__bm_sync_locks` 객체에 `{[blogId]: ts}`. 5분 이상 좀비 락은 강제 해제(SW 재시작 시 stale 락 방지).
- 1회 sync 최대 10개 글(`POSTS_PER_SYNC`).

### 3.5 chrome.alarms 스케줄러
- 알람명: `benchmark.syncAll`
- 첫 발화: 부팅 5분 후 (부팅 직후 폭주 방지)
- 주기: 180분 (3시간)
- `ensureBenchmarkAlarm()` idempotent — `onInstalled` 와 SW boot 양쪽에서 호출 가능.
- `onAlarm` 발화 시 현재 Supabase 세션의 user.id 확인 → 미로그인이면 조용히 skip(알람은 유지).

### 3.6 manifest.json 권한 변경
- 추가: `"alarms"` (1개)
- 유지: `storage`, `sidePanel`, `scripting` + 기존 host_permissions 3개(blog.naver.com 변경 없음).
- 최소 권한 원칙 준수 — `<all_urls>` / activeTab 등 위험 권한 0건.

## 4. 보안·안전 체크포인트

| 항목 | 적용 방식 |
|---|---|
| host_permissions 외 fetch | `ALLOWED_HOSTS` Set 으로 양 collector 에서 검증 → 위반 시 throw |
| XSS (content_snippet) | `stripHtml()` 로 저장 전 평문화 |
| innerHTML 류 0건 | `check-dom-unsafe.sh` PASS — 양 collector 모두 DOM 에 안 붙임 |
| sender 출처 검증 | service-worker 의 기존 `isTrustedSender()` 가 모든 경로에 적용 |
| 동적 코드 실행 | 없음 (collector 는 정규식·DOM 트리 순회만) |
| API 키 노출 | 본 작업 0건. qa-script 오탐 1건은 supabase TS 주석 — 필터 확장으로 해결 |
| 알람 권한 추가 후 회귀 | check-rls-enabled / check-email-admin 무관, 5/5 PASS |
