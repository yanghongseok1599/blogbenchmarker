## 중요: 기존 benchmark-repo.js 보존

benchmark-repo.js 가 이미 작성되어 있었다 (`addBlog(userId, blogUrl, blogName)` 포지셔널 시그니처 + `upsertPosts` 등 sync 용 API 포함). 덮어쓰지 않고 **추가만** 했다:

- 기존 API 그대로: `addBlog` / `removeBlog` / `listBlogs` / `getBlog` / `listAllForSync` / `upsertPosts`

benchmark-tab.js 는 기존 포지셔널 시그니처(`addBlog(userId, blogUrl, blogName)`)에 맞춰 호출.

## 기능 흐름

### 1) 마운트
```text
panel.js : 탭 클릭 → tabActivators.benchmark(panelEl)
    ↓
benchmark-tab.js:mountBenchmarkTab
    ├─ DOM 조립 (createEl, innerHTML 미사용)
    ├─ getCurrentUser()           → 로그인 없음 시 폼 비활성 + 에러
    ├─ listBlogs(userId)          → 캐시 map 초기화
    ├─ renderList()
    ├─ subscribeToChanges(userId, onEvent) → Realtime 채널 오픈
    └─ window.beforeunload → cleanup (unsubscribe)
```

### 2) 추가
```text
form submit
    ↓
parseNaverBlogUrl(raw)   [클라이언트 1차 검증/정규화]
    ├─ invalid → showStatus("네이버 블로그 URL 형식이 아닙니다", error)
    └─ valid   → addBlog(userId, canonicalUrl, blogId)
                    ├─ 23505 중복 → "이미 등록된 블로그입니다" (repo 측 변환)
                    └─ success   → blogById.set() + renderList()
```

### 3) 삭제
```text
li > button[data-action="benchmark-remove"] click
    ↓
removeBlog(id) → RLS 가 본인만 허용
    ↓
blogById.delete(id) + renderList()
```

### 4) Realtime 동기화
```text
다른 기기/세션이 INSERT/UPDATE/DELETE 발생
    ↓
Postgres Changes → supabase.channel(...).on('postgres_changes', {user_id=eq.{userId}})
    ↓
handleRealtimeEvent({ event, new, old })
    ├─ DELETE → blogById.delete(old.id)
    └─ INSERT/UPDATE → blogById.set(new.id, new)
renderList()   [sort by added_at DESC]
```

## url-parser.js 상세

### 지원 포맷 (6종)
| 포맷 | 예시 | 결과 |
|---|---|---|
| 홈(PC) | `https://blog.naver.com/example` | blogId=example, postId=null |
| 글(PC) | `https://blog.naver.com/example/223456789` | blogId=example, postId=223456789 |
| PostView | `https://blog.naver.com/PostView.naver?blogId=example&logNo=223456789` | blogId=example, postId=223456789 |
| 홈(모바일) | `https://m.blog.naver.com/example` | blogId=example, postId=null |
| 글(모바일) | `https://m.blog.naver.com/example/223456789` | blogId=example, postId=223456789 |
| PostView(모바일) | `https://m.blog.naver.com/PostView.naver?...` | (동일) |

### 방어 로직
- 프로토콜 누락 시 `https://` 자동 보정 1회 시도 (`blog.naver.com/foo` 같은 붙여넣기 UX).
- `RESERVED_FIRST_SEGMENTS` (`PostView.naver`, `PostList.naver`, `prologue`)를 blogId 로 오인하지 않음.
- `BLOG_ID_RE = /^[A-Za-z0-9_-]{1,30}$/`, `POST_ID_RE = /^\d{1,20}$/` 로 엄격 검증.
- `canonicalUrl` 은 항상 **홈 URL** (`https://blog.naver.com/{blogId}`) — 즐겨찾기 중복 방지(`UNIQUE(user_id, blog_url)`)에 기여.

### export
- `isNaverBlogUrl(raw): boolean`
- `parseNaverBlogUrl(raw): { isValid, blogId, postId, canonicalUrl }`
- `normalizeBlogHomeUrl(raw): string | null`

## benchmark-repo.js 확장 API

### listPosts(blogId, opts?)
- fetched_at DESC, default limit=50, max=200.
- benchmark_posts 접근은 RLS 가 `benchmark_blogs.user_id = auth.uid()` 간접 검증.

### subscribeToChanges(userId, onChange)
- `supabase.channel('benchmark_blogs:{userId}:{timestamp}')` 고유 이름.
- 서버측 filter: `user_id=eq.{userId}` — 타인 변화 수신 0.
- **반환값은 async unsubscribe 함수** — 호출 측이 반드시 보관하고 정리해야 누수 없다.
- 콜백 payload: `{ event: 'INSERT'|'UPDATE'|'DELETE', new, old }` (Supabase 원본 eventType 을 정규화).

```js

const panel = document.querySelector('[data-panel="benchmark"]')
await mountBenchmarkTab(panel)

// 컴포넌트 제거 시

```

### Realtime 단독 사용 (다른 탭에서)

```js
import { subscribeToChanges } from '../lib/repositories/benchmark-repo.js'

const unsub = subscribeToChanges(userId, ({ event, new: row, old }) => {
  console.log(event, row ?? old)
})

// 정리
await unsub()
```

### URL 파싱 단위

```js
import { parseNaverBlogUrl } from '../lib/utils/url-parser.js'

parseNaverBlogUrl('blog.naver.com/example')
// { isValid: true, blogId: 'example', postId: null, canonicalUrl: 'https://blog.naver.com/example' }

parseNaverBlogUrl('https://blog.naver.com/PostView.naver?blogId=foo&logNo=12345')
// { isValid: true, blogId: 'foo', postId: '12345', canonicalUrl: 'https://blog.naver.com/foo' }

parseNaverBlogUrl('https://tistory.com/bar')
// { isValid: false, blogId: null, postId: null, canonicalUrl: null }
```
