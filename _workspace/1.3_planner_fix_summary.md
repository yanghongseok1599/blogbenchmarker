## 주요 설계 결정

### supabase-client.js
- **UMD 글로벌 소비**: `@supabase/supabase-js` npm import 가 아닌 `globalThis.supabase.createClient` 를 요구한다. 이유: MV3 CSP(`script-src 'self'`)는 원격 CDN 을 차단하므로 UMD 번들은 **로컬에 벤더링** 되어야 한다 (예: `lib/vendor/supabase.umd.min.js`). 래퍼는 글로벌 부재 시 명확한 에러를 던져 벤더링 누락을 즉시 노출한다.
- **chrome.storage.local adapter**: Service Worker idle 재시작/사이드패널 재오픈 시에도 세션이 복원되도록 `storage` 어댑터 필수. `chrome-extension-security §2-3` 반영.
- **`detectSessionInUrl: false`**: 확장프로그램은 URL hash 기반 OAuth redirect 경로가 없으므로 비활성.
- **env 템플릿 가드**: `your-project-ref` / `YOUR_KEY_HERE` 감지 시 `console.warn` (throw 금지 — 서비스 워커 죽이면 메시지 라우터 멈춤).

### dom-safe.js
- **금지 API 0회 사용**: 파일 내부에서도 innerHTML/outerHTML/insertAdjacentHTML 금지. 자가 준수 검증 가능 (`grep`).
- **`createEl(tag, attrs, children)`**: `className`, `style`(객체), `onClick` 이벤트 리스너, `data-*` 속성 지원. 문자열 자식은 자동 `TextNode` 변환 → HTML 해석 불가.
- **`script` 태그 생성 차단**: `createEl('script', ...)` 호출 시 명시적 throw.
- **`escapeHtml(str)`**: 7개 특수문자(`& < > " ' / \``) 이스케이프. 최후의 수단이며, `safeText`/`createEl` 을 우선 쓰도록 JSDoc 에 명시.

### user-repo.js
- **필드 화이트리스트**: `UPDATABLE_FIELDS = ['display_name']` 만 허용. `plan` 은 `verify-subscription` Edge Function, `is_admin` 은 `admin-actions` 경로에서만 변경. 클라이언트 위변조 원천 차단.
- **`isAdmin` 안전 기본값**: 조회 에러 시 `false` 반환. "확정 true" 만 관리자로 인정해 권한 에스컬레이션 차단.
- **RLS 신뢰**: 본인 외 userId 조회는 RLS 가 0행/에러 반환 → 클라이언트에서 추가 권한 체크 없음. 실패 메시지는 사용자 친화로 래핑(`'프로필 조회 실패: ...'`).
- **changelog 정합성**: 선택 컬럼은 `id, email, display_name, plan, is_admin, created_at` 6개 (changelog §1.1 과 일치). `language`/`updated_at` 은 현 스키마에 없어 제외 — QA 리포트의 HIGH 건(F-4, B-9) 해소 시 이 repo 도 확장 필요.

## 사용 예제

### 1) background/handlers/auth-handler.js 에서

```js
import { getCurrentUser, supabase } from '../../lib/supabase-client.js'
import { getProfile } from '../../lib/repositories/user-repo.js'

export const authHandler = {
  async status() {
    const user = await getCurrentUser()
    if (!user) return { loggedIn: false }
    const profile = await getProfile(user.id)
    return { loggedIn: true, profile }
  },
  async logout() {
    await chrome.storage.local.set({ __intentional_logout: true })
    const { error } = await supabase.auth.signOut()
    if (error) throw error
    await chrome.storage.local.clear()
    return { redirectTo: 'auth/login.html' }
  },
}
```

### 2) sidepanel/tabs/mypage-tab.js 에서 (DOM 안전 렌더)

```js
import { createEl, clearAndAppend, safeText } from '../../lib/dom-safe.js'
import { getCurrentUser } from '../../lib/supabase-client.js'
import { getProfile, updateProfile } from '../../lib/repositories/user-repo.js'

export async function renderMypage(container) {
  const user = await getCurrentUser()
  if (!user) {
    clearAndAppend(container, createEl('p', {}, '로그인이 필요합니다.'))
    return
  }
  const profile = await getProfile(user.id)

  const nameInput = createEl('input', {
    type: 'text',
    value: profile?.display_name ?? '',
    className: 'bm-input',
  })
  const saveBtn = createEl('button', {
    className: 'bm-btn',
    onClick: async () => {
      const updated = await updateProfile(user.id, { display_name: nameInput.value })
      safeText(statusEl, `저장됨: ${updated.display_name}`)
    },
  }, '저장')
  const statusEl = createEl('p', { className: 'bm-status' })

  clearAndAppend(container,
    createEl('h2', {}, '마이페이지'),
    createEl('p', {}, `플랜: ${profile?.plan ?? 'free'}`),
    createEl('label', {}, '닉네임', nameInput),
    saveBtn,
    statusEl,
  )
}
```

### 3) 관리자 가드 (background/handlers/admin-handler.js 에서)

```js
import { getCurrentUser } from '../../lib/supabase-client.js'
import { isAdmin } from '../../lib/repositories/user-repo.js'

export async function requireAdmin() {
  const user = await getCurrentUser()
  if (!user) throw new Error('로그인이 필요합니다.')
  if (!(await isAdmin(user.id))) throw new Error('관리자 권한이 필요합니다.')
  return user
}
```
