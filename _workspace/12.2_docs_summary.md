## 설계 결정

### 1. 법률 준수
- **개인정보 처리방침**: 한국 개인정보보호법(PIPA) 제35~39조 + EU GDPR 제6·15~22조 양쪽 요건 동시 충족. 수집 항목을 표로 분해해 각 항목의 **법적 근거(contract/consent/legitimate interest)** 를 명시. 국외 이전 고지, 아동(만 14세) 조항, 5년 결제 기록 보관 의무(전자상거래법) 모두 반영.
- **이용약관**: 7일 이내 청약 철회권 명시, 전액/부분/불가 구분. 회사 귀책 장애 시 전액 환불. 관할 법원은 서울중앙지방법원 전속관할.
- **과장 금지**: "무조건 1위", "수익 보장" 등 금지 표현을 STORE-LISTING §1의 금지 사항에 명문화. 본문에도 "AI 생성물의 정확성·검색 순위 향상·수익 발생은 보장하지 않습니다" 고지 포함.

### 2. Placeholder 규칙
실제 값이 결정되기 전 단계이므로 `{{FOO}}` 형식의 이중 중괄호 placeholder 사용. 프로덕션 배포 전 일괄 치환이 필요합니다:

### 3. Chrome Web Store 특화
- `short_description`은 132자 한도 준수. 영어 초안이 135자여서 103자 축약본을 병기 제안.
- `detailed_description` 한국어 마스터는 약 1,650자 — 15,000자 한도에 충분히 여유.
- 권한별 **공개 사유를 한/영 병기** — 스토어 제출 폼에 바로 복붙 가능.
- **Single Purpose 선언**: "Analyze, benchmark, and draft Naver blog posts." — MV3 심사의 필수 항목.
- **Remote code: No** 명시 — CSP `script-src 'self'` 유지와 일치.

### 4. 스크린샷 가이드
- 5종 구성을 실제 앱 탭 순서(분석/벤치마크/생성/마이/도구)에 맞춰 제안.
- 각 캡션을 **한/영/일 3개 언어** 로 병기 — `_locales` 3개 언어 지원과 일치.
- 개인 이용자 글 캡처 시 사전 서면 동의 요구 — 법적 리스크 차단.
- 기기 목업·Chrome UI 조작 금지 원칙 명문화 — 스토어 reality manipulation 정책 준수.

### 5. 사용자 매뉴얼
- 플로우를 **설치→가입→분석→벤치마크→AI→학습→YouTube→도구→마이→트러블슈팅** 10단계로 분할.
- 각 기능에 스크린샷 placeholder(`docs/img/01-install.png` 등) 삽입 — Phase 12.2 이미지 생성 시 파일명과 1:1 매칭.
- 점수 해석 표에 "검색 결과 순위를 보장하지 않습니다" 경고 삽입.
- 고객 문의 시 **비밀번호·결제 정보 첨부 금지** 가이드 포함.

### 6. CHANGELOG
- Keep a Changelog 표준(Added/Changed/Fixed/Security) 준수.
