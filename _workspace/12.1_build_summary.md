## 빌드 단계 (build.sh)

```
(1) dist/ 완전 제거 + dist/extension 생성
(2) rsync -a extension/ → dist/extension/
    제외: lib/env-config.js, lib/env-config.local.js, .DS_Store, *.map, *.md, *.log,
          __tests__/, node_modules/, *.test.js
    안전장치: dist/ 에 env-config.js 존재 시 즉시 실패
(3) node scripts/strip-console.js dist/extension
    → console.{log,warn,info,debug,trace} 제거, console.error 유지
(4) package.json.version → dist/extension/manifest.json.version
(5) scripts/package-extension.sh $VERSION → dist/blogbenchmarker-$VERSION.zip
(6) 요약 (파일 수, 총 크기, zip 크기)
```

## strip-console.js 설계 결정

### 정규식 기반 (AST 미사용) 이유
- **devDependencies 0 목표** — acorn/esbuild 등 파서 추가 시 `node_modules` 필수
- MV3 확장 코드 패턴 단순 — class/arrow 만 주로 사용, 복잡한 JSX/TS 없음

### 한계 + 방어 로직
1. **체이닝 감지**: `console.log(x).then(...)` 같은 체이닝이 뒤에 붙으면 제거하지 않음(문법 손상 방지). README Q&A 에 명시.
2. **문자열/주석 내부 보존**: `findStringEnd` / `findTemplateEnd` / 블록·라인 주석 스킵 — 로그 문구 속 `"console.log"` 문자열은 유지.
3. **균형 괄호**: `findMatchingParen` 이 문자열/템플릿/주석을 인지하며 nested paren 을 정확히 매칭.
4. **문장 단독 vs 표현식**: 단독 statement 면 앞쪽 빈 줄과 뒤 개행까지 소비 → 빈 줄 누적 방지. 표현식 문맥(조건식 안)에 있으면 보수적 처리.
5. **Word boundary**: `console` 앞이 영숫자/_/$ 면 `myconsole.log` 같은 오탐 회피.

### 환경변수
- `STRIP_CONSOLE_DRY=1` → 파일 수정 없이 로그만 출력 (CI 프리뷰 용도).

## verify-build.sh 체크포인트

| 코드 | 항목 | 실패 분류 |
|---|---|---|
| A | 필수 파일 존재 (manifest, service-worker, panel, supabase-client, dom-safe, env-config.example, icons/README) | MEDIUM |
| B | `env-config.js` 미포함 + `env-config.example.js` 포함 | **BLOCKER** |
| C | `console.log/warn/info/debug/trace` 잔존 0건 (주석 제외) | HIGH |
| D | `manifest.version == package.version` | HIGH |
| E | 시크릿 하드코딩 패턴 0건 (`AIza...`, `sk_live_`, `sb_secret_`, `service_role`, `whsec_`, `TOSS_SECRET_KEY=`, `PORTONE_API_SECRET=`) | **BLOCKER** |
| F | `innerHTML` 할당 0건 | **BLOCKER** |

exit code 0/1 로 CI/CD 연결 가능.

## 규칙 준수 체크

- [x] **console.error 유지** — `STRIPPED_METHODS` 에 `'error'` 미포함 (strip-console.js:31). verify-build 에서도 `console.error` 건수를 참고 출력만.
- [x] **macOS + Linux 호환** — `rsync`, `zip`, `du -sh`, `awk`, `node` 만 사용. `brew install gnu-*` 불필요. 의존성 부재 시 스크립트 상단에서 명시적 실패.
- [x] **dist/ 는 `.gitignore`** — 이미 존재(.gitignore:19). `*.zip` 도 이미(.gitignore:42).
- [x] **`set -euo pipefail`** — 3개 쉘 스크립트 전부. 에러 즉시 종료.
- [x] **한국어 로그 OK** — `[build]`, `[strip]`, `[verify]`, `(1/6) dist/ 정리` 등 혼용.
- [x] **금지 디렉토리 미수정** — `extension/`, `supabase/`, `_workspace/qa-scripts/` 건드리지 않음. `.gitignore` 는 수정 가능 대상이었으나 이미 필요 규칙 충족 상태라 손대지 않음.
- [x] **devDependencies 비움** — `package.json.devDependencies` 필드 자체 부재. `node` + `bash` + `rsync` + `zip` 만으로 동작.

## 사용 예제

### 전체 파이프라인
```bash
npm run all
# = npm run build && npm run verify
```

### 개별 단계
```bash
npm run build             # dist/ 생성 + zip
npm run verify            # 6개 체크포인트 검증
npm run package           # 이미 빌드된 dist/extension → 재패키징
npm run strip-console     # 특정 경로에 수동 적용
```

### Dry-run (로그만)
```bash
STRIP_CONSOLE_DRY=1 node scripts/strip-console.js dist/extension
```

### 커스텀 버전
```bash
# package.json 수정 없이 zip 버전만 오버라이드
bash scripts/package-extension.sh 1.2.3-rc.1
```
